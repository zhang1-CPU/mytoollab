Understanding Different Types of QR Codes

All QR codes store data, but the data format determines what A free QR code types tool is essential — a phone does with it. Here is a guide to URLs, vCards, WiFi, email, SMS, geo-location, and more, with practical use cases for each.

Static vs. Dynamic QR Codes

A QR code is simply an image that encodes text. Two broad categories cover every use case.

  • Static QR codes encode the target information directly into the pattern. The content cannot be changed without regenerating and reprinting the code. These are free, simple, and permanent.
  • Dynamic QR codes store a short URL that redirects to the actual content. Because the redirection happens server-side, you can change what the URL points to without changing the printed code. Useful for marketing campaigns where you want to measure scans or update the landing page later.

Dynamic codes need a tracking service and are slightly more complex. For most everyday uses — a business card, a menu, a WiFi password — a static QR code is perfect.

The Data Types Your Phone Recognizes

Modern phones interpret certain patterns inside a QR code automatically. The pattern decides the action: open a URL, add a contact, connect to WiFi, and so on.

1. URL / Website

The most common QR code by a wide margin. Your phone opens the link in the default browser.

https://example.com/summer-menu
  • Best for: marketing posters, product packaging, business cards pointing to a landing page.
  • Tip: keep the URL short. Every additional character adds more modules to the QR code, making it harder to scan.

2. Plain Text

A plain-text QR code shows the text on screen with no automatic action. Useful for messages, notes, or information that does not need automation.

Pickup code A-215. Thank you!

3. vCard — Contact Information

A vCard QR code encodes structured contact data so the phone prompts "Add to contacts." The formal format begins with BEGIN:VCARD.

BEGIN:VCARD
VERSION:3.0
N:Doe;John
ORG:Acme Inc.
TEL:+1-555-0199
EMAIL:john@example.com
URL:https://example.com
END:VCARD
  • Best for: business cards, conference badges, storefront windows.
  • Note: support for vCard fields varies by phone. Stick to the basics (name, phone, email, organization) for widest compatibility.

4. WiFi Configuration

A WiFi QR code encodes network credentials so a phone can connect with a single tap. This is the standard format used by Android and iOS camera apps.

WIFI:T:WPA;S:HomeNetwork;P:secret123;;

Field breakdown: T is the encryption type (WPA, WEP, or "nopass" for open networks), S is the network name (SSID), P is the password.

  • Best for: cafes, offices, co-working spaces, vacation rentals.
  • Security: Anyone can scan and read the password. Place it where you already trust the audience. Rotate guest-network passwords monthly.

5. Email (mailto)

Scanning launches the default email app with recipient, subject, and body pre-filled.

mailto:hello@example.com?subject=Inquiry&body=Hello%2C%20
  • Best for: customer-support posters, product registration, "email us" kiosks.

6. SMS / Text Message

Opens the SMS app with a phone number and optional message pre-filled.

SMSTO:+15550199:Text us for a free sample

7. Telephone Number

Prompts the phone to make a call.

tel:+15550199

Useful on window signs ("Call for a free estimate") and emergency-information posters.

8. Geo-Location

Opens a map at a specific latitude and longitude.

geo:40.7128,-74.0060
  • Best for: "Find us" posters, real-estate flyers, travel guides.
  • Tip: pull coordinates from Google Maps by right-clicking a spot.

9. Event / Calendar

Adds an event to the phone's calendar with title, location, start time, and end time.

BEGIN:VEVENT
SUMMARY:Design Review
DTSTART:20260715T100000Z
DTEND:20260715T110000Z
LOCATION:Conference Room 3
END:VEVENT

Size, Error Correction, and Scannability

QR codes come in 40 sizes (versions). Version 1 is a 21×21 module pattern that can store a short URL. Version 40 is 177×177 and can store thousands of characters. In practice, versions 1 through 10 handle most real-world uses.

Error correction is the percentage of the pattern that can be damaged while still remaining scannable. Four levels are available:

LevelRecoveryTypical use
L~7%Web URLs, indoor use
M~15%General-purpose, business cards
Q~25%Product labels, outdoor posters
H~30%Labels that may get damaged

Higher error correction means more modules for the same amount of data — so the QR code physically grows. Pick the lowest level that matches where the code will live.

Scannability tip. Leave a quiet zone (blank white margin) at least four modules wide around every QR code. Phones need this margin to detect where the pattern begins and ends. Without it, tight designs fail to scan.

Best Practices

  • Keep content short. A shorter QR code is easier to scan from a distance and holds up better when printed small. Use short URLs when possible.
  • Test on both Android and iOS. The two platforms have subtle differences in how they prompt actions (especially WiFi and vCard).
  • Prefer HTTPS URLs. Modern browsers warn on plain HTTP, which hurts trust and conversion.
  • Provide a text fallback. Next to a WiFi QR code, also print the network name and password for devices that cannot scan.
  • Choose colors with enough contrast. The modules must be darker than the background. Dark blue on white works; red on pink does not. If you must brand a QR code, use level H error correction so small visual changes do not break it.
  • Do not put sensitive data in a QR code without a reason. Anyone can scan and read it. WiFi passwords, internal URLs, and personal information should live behind a login or on a guest network, not in a static QR code posted publicly.

Common Pitfalls

  • URLs with special characters. Ampersands, question marks, and non-English characters must be encoded properly. A good QR generator does this for you; double-check if you craft them by hand.
  • Long vCards failing to scan. If a vCard contains a physical address, two phone numbers, a work email, a personal email, and a title, it may overflow into a version the phone struggles with. Trim fields you do not need.
  • Printed too small. If modules are smaller than about 3 mm on paper, many phones fail from arm's length. Target 10 mm per module for outdoor posters.
  • Glossy or curved surfaces. Reflective paper and curved labels cause glare. Use matte stock and flat surfaces where possible.
One last test. Before committing to a print run, scan the final design from the actual distance users will stand. If it fails, increase the size, boost error correction, or shorten the content.

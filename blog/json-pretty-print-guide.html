JSON Pretty Print: How to Format JSON for Better Readability

Ever stared at a wall of minified JSON trying to find one misplaced key? You are not alone. Raw JSON is designed for computers to parse, not for humans to read. JSON pretty print transforms that unreadable blob of text into nicely formatted, indented data you can actually understand. Learn how to format JSON instantly using online tools, browser dev tools, command line utilities, and more.

What Is JSON Pretty Print?

JSON (JavaScript Object Notation) is the most common data format for APIs, config files, and web applications. By default, JSON is often minified — all whitespace is removed to make it as small as possible for transmission. The result is efficient for computers but completely unreadable for humans.

JSON pretty print is the process of adding indentation, line breaks, and spacing to JSON data so humans can read and understand it more easily. It does not change the data at all — it only changes how it is displayed.

Compare this minified JSON:

{"name":"John","age":30,"city":"New York","hobbies":["reading","gaming","hiking"],"address":{"street":"123 Main St","zip":"10001"}}

To this pretty-printed version:

{
  "name": "John",
  "age": 30,
  "city": "New York",
  "hobbies": [
    "reading",
    "gaming",
    "hiking"
  ],
  "address": {
    "street": "123 Main St",
    "zip": "10001"
  }
}

The data is exactly the same, but the second version you can actually scan and understand at a glance.

Why JSON Formatting Matters

You might be wondering — if JSON is for computers, why bother making it readable? Here are a few situations where pretty JSON output saves you time and frustration:

  • Debugging API responses. When an API returns unexpected data, you need to see the structure clearly to find what is wrong.
  • Reviewing config files. JSON configuration files are much easier to edit when properly formatted.
  • Sharing data with teammates. Formatted JSON is easier to discuss and review in pull requests or documentation.
  • Learning JSON structure. When you are new to JSON, proper indentation helps you understand how objects and arrays nest together.
  • Finding errors. A misplaced comma or bracket is much easier to spot in formatted JSON.

5 Ways to Make JSON Readable

1. Online JSON Formatter

The fastest and easiest way to format JSON is using a JSON formatter online tool. Just paste your JSON, click a button, and get beautifully formatted output. No installation, no command line, no hassle.

Look for a JSON beautifier that:

  • Works entirely in your browser (your data never leaves your device)
  • Lets you adjust indentation level (2 spaces, 4 spaces, tabs)
  • Highlights syntax with colors
  • Validates JSON and shows helpful error messages
  • Supports minifying (the opposite of pretty printing)

This is the best approach for quick formatting tasks, especially when you are working on someone else's computer or do not have your dev tools set up.

2. Browser Developer Tools

If you are debugging an API call in your browser, you do not even need to leave the browser. Modern browsers have built-in JSON pretty printing in their developer tools.

In Chrome, Firefox, or Edge:

  1. Open DevTools (F12 or right-click and choose Inspect)
  2. Go to the Network tab
  3. Click on the request you want to inspect
  4. Go to the Response tab
  5. The browser automatically formats JSON responses

You can also format JSON directly in the Console tab using JSON.stringify() with the third parameter for indentation:

const data = { name: "John", age: 30 };
console.log(JSON.stringify(data, null, 2));

3. Command Line Tools

If you work in the terminal, there are several great tools for formatting JSON output from APIs or files.

jq is the most popular. It is a lightweight and flexible command-line JSON processor. Once installed, you can pipe JSON through it for instant formatting:

curl https://api.example.com/data | jq .

python -m json.tool is always available if you have Python installed:

cat data.json | python -m json.tool

node -e works if you have Node.js:

cat data.json | node -e "console.log(JSON.stringify(JSON.parse(require('fs').readFileSync(0,'utf8')), null, 2))"

4. Code Editors and IDEs

Every modern code editor has built-in JSON formatting or an extension that does it. In VS Code, for example, you can right-click in a JSON file and choose "Format Document" (or press Shift+Alt+F on Windows, Shift+Option+F on Mac).

Most editors also have extensions like Prettier that can auto-format JSON (and other languages) every time you save a file. This is a game-changer for working with JSON config files on a daily basis.

5. Programmatically in Code

If you are a developer, you can pretty print JSON directly in your code using built-in functions. This is useful for logging, debugging, and generating readable output.

In JavaScript/Node.js:

JSON.stringify(data, null, 2);
// The third argument sets the indentation level

In Python:

import json
json.dumps(data, indent=2)

In Go:

import "encoding/json"
json.MarshalIndent(data, "", "  ")
Pro tip: The second argument to JSON.stringify in JavaScript is a replacer function. You can use it to filter or transform values before they are serialized. For most pretty-printing purposes, just pass null.

Understanding JSON Indentation

When you format JSON, you have choices about how to indent it. The most common options are:

  • 2 spaces: The most common convention, especially in JavaScript and web development. Compact but still readable.
  • 4 spaces: More common in Python and some other languages. Easier to see nesting levels but uses more horizontal space.
  • Tabs: Less common for JSON but used in some codebases. Lets the reader adjust the display width.

There is no right answer — it mostly comes down to personal preference and team conventions. The important thing is to be consistent. Pick one style and stick with it throughout a project.

JSON Pretty Print vs. Minify

Pretty printing and minifying are opposites, but both are useful. You will often find yourself going back and forth between them.

Pretty print: Add whitespace and indentation for readability. Use during development, debugging, and code review.

Minify: Remove all unnecessary whitespace to make the JSON as small as possible. Use in production APIs, config files where size matters, and for network transmission.

A good JSON formatter tool will let you do both — format JSON for reading and minify it for production. Having both functions in one tool saves you from switching between tools constantly.

Common JSON Formatting Mistakes

  • Trailing commas. Unlike JavaScript objects, JSON does not allow trailing commas after the last item in an object or array. This is one of the most common JSON syntax errors.
  • Single quotes. JSON requires double quotes for both keys and string values. Single quotes are not valid JSON.
  • Unquoted keys. In JSON, all object keys must be quoted. Unlike JavaScript, you cannot use unquoted identifiers as keys.
  • Comments. JSON does not support comments. If you need comments, use a format like JSON5 or YAML instead.
  • Tabs vs. spaces wars. While both are technically valid for indentation, mixing them in the same file is a bad idea. Pick one and be consistent.

Advanced: JSON and CSV Conversion

Sometimes you need to convert JSON to another format entirely. CSV (Comma-Separated Values) is one of the most common conversions, especially when working with spreadsheets or data analysis tools.

A JSON to CSV converter can transform structured JSON data into flat CSV format that you can open in Excel, Google Sheets, or any data analysis tool.

This is incredibly useful for: exporting API data to spreadsheets, analyzing JSON data with pivot tables, sharing data with non-technical stakeholders, and importing data into databases.

Frequently Asked Questions

What is JSON pretty print?

JSON pretty print is the process of formatting JSON data with proper indentation, line breaks, and spacing to make it human-readable. It does not change the data itself — only how it is displayed. The opposite of pretty printing is minifying, which removes all whitespace to make the JSON as small as possible.

How do I make JSON readable?

The easiest way is to use a JSON beautifier or formatter tool. Simply paste your unformatted JSON into the tool and it will output nicely indented, readable JSON. You can also use browser developer tools, command-line tools like jq, or your code editor's built-in formatting function.

How do I format JSON in Chrome?

Chrome automatically formats JSON responses in the Network tab of DevTools. You can also use the Console to format JSON with JSON.stringify(data, null, 2). For quick formatting of arbitrary JSON text, use a browser-based JSON formatter website.

What is the best indentation for JSON?

2-space indentation is the most common convention in web development, but 4 spaces and tabs are also used. The "best" option depends on your team's style guide and personal preference. What matters most is consistency — pick one style and use it throughout your project.

Can pretty-printed JSON be used in production?

Yes, it is still valid JSON — it just has extra whitespace. However, pretty-printed JSON is larger and slower to transfer, so it is generally not recommended for production APIs. Minify your JSON for production to reduce file size and improve performance.

What is Base64?

Base64 is a binary-to-text encoding scheme that converts any sequence of bytes into a human-readable ASCII string using 64 characters (A–Z, a–z, 0–9, +, /, with = for padding). It is widely used for embedding images in HTML/CSS, passing data through URLs, and storing binary content inside text protocols like email.

Use cases

  • Embedding small images directly in HTML or CSS files as data URIs.
  • Encoding API credentials for safe transmission via HTTP Basic auth headers.
  • Decoding Base64 payloads found in JWT tokens, emails, or logs.
  • Encoding or decoding attachments in email (MIME) or JSON payloads.

Frequently Asked Questions

Is Base64 encryption?

No. Base64 is encoding, not encryption — it provides no confidentiality. Anyone can reverse it. Use AES or similar for sensitive data.

Why does my decoded output look garbled?

The input may encode binary data (an image, a PDF, a ZIP file). Use the Download button to save it as a binary file instead of copying text.

Does Base64 increase file size?

Yes, by approximately 33% (each 3 bytes become 4 characters + padding).

Is my data sent to a server?

No. Everything runs locally in your browser using native Web APIs. Nothing you paste or upload is transmitted anywhere.

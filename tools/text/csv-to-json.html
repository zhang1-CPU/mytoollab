How it works

  1. Paste or upload your data. CSV → JSON respects quoted fields, embedded commas, escaped double-quotes (""), and multi-line cells.
  2. JSON → CSV accepts an array of flat objects and optionally writes a header row using the keys of the first object.
  3. Results can be copied to clipboard or downloaded as a file. Everything runs locally in your browser.

CSV parsing rules

This converter follows RFC 4180 with a small number of practical extensions: fields can be quoted with double-quotes, double-quotes inside quoted fields are escaped by doubling them (""), embedded newlines and commas inside quoted fields are preserved, and line endings (CR, LF, CRLF) are all accepted.

Use cases

  • Converting spreadsheet exports to JSON for API endpoints.
  • Turning API results into CSV for Excel or Google Sheets.
  • Cleaning CSV files with inconsistent quoting or line endings.
  • Previewing JSON as rows before database or BI import.

Frequently Asked Questions

My CSV has multiple-line cells. Will that work?

Yes. Fields wrapped in double-quotes can contain newlines, commas, and escaped double-quotes ("").

What JSON shapes are supported for JSON to CSV?

The converter expects a flat array of objects (e.g. [{"name":"Alice"}]). Deeply nested objects are flattened using their JSON representation.

Is my data uploaded anywhere?

No. All conversion happens in your browser using JavaScript only.

What is URL encoding?

URL encoding (also called percent-encoding) converts special characters outside the safe ASCII set into sequences like %20 (space) and %26 (ampersand). URLs can only contain letters, digits, and a small set of punctuation — everything else must be percent-encoded to be transmitted safely.

Use cases

  • Encoding URL query parameters with spaces, Unicode, and punctuation.
  • Decoding URLs from browser address bars and server logs.
  • Debugging JWT tokens and OAuth parameters embedded in URLs.
  • Preparing search queries or form data for API endpoints.

Frequently Asked Questions

What's the difference between encodeURI and encodeURIComponent?

encodeURIComponent encodes everything that isn't a letter, digit, or - _ . ~ ! ' ( ) * — ideal for parameter values. encodeURI keeps URL-safe characters like : / ? & = # intact and is used for full addresses.

Why do spaces become + in some places?

Inside application/x-www-form-urlencoded bodies (like HTML forms), spaces are traditionally encoded as + for historical reasons. The browser standard, however, encodes spaces as %20 in URLs.

Is my data sent to a server?

No. All encoding and decoding happens in your browser using native JavaScript functions.

Color Basics for Designers and Content Creators

Every website, presentation, and app depends on color — but how does it actually work in code? Here is a practical tour of HEX, RGB, and HSL: what each one means, when to use each, and how to pick colors that look good together.

How Computers Think About Color

Human eyes perceive light in three bands — roughly red, green, and blue. Digital displays mirror that: every pixel on a screen is three tiny sub-pixels, one red, one green, one blue. Everything else is a mix.

That simple fact is why every color format you encounter on the web is ultimately a way to describe those three channels plus an optional fourth: transparency (the alpha channel).

HEX: The Six-Character Shorthand

HEX is the most common color format in HTML and CSS. It uses the hexadecimal number system (base 16, digits 0–9 and letters a–f) to describe red, green, and blue channels in exactly six characters.

#2563eb  →  red: 0x25 = 37, green: 0x63 = 99, blue: 0xeb = 235

Eight-character HEX includes alpha (transparency): the last two characters describe opacity from 00 (fully transparent) to ff (fully opaque).

  • Best for: Brand colors, CSS variables, copy-pasting from design tools, any color that does not need mental math.
  • Common patterns: Black is #000000, white is #ffffff, and grays sit along the diagonal where all three channels are equal (e.g. #64748b).
  • Shorthand: #226 is a shorter way to write #222266. Valid only when each pair of digits is repeated.

RGB and RGBA: The "Talk in Numbers" Format

RGB accepts three values from 0 to 255, one per channel. RGBA adds a fourth value — alpha — between 0.0 and 1.0 for transparency.

color: rgb(37, 99, 235);    /* opaque blue */
color: rgba(37, 99, 235, 0.5); /* same blue, 50% see-through */
  • Best for: Values pulled from design tools; dynamic colors generated in JavaScript; transparency on hover states and overlays.
  • Modern CSS shortcut: You can now write rgb(37 99 235 / 0.5) without commas or the "a" — this is increasingly common in new code.

HSL: The Format That Actually Feels Human

HSL stands for Hue, Saturation, Lightness. It is the only common format where the numbers map to how humans think about color:

  • Hue (0–360): The angle on a color wheel. 0 and 360 are both red; 120 is green; 240 is blue.
  • Saturation (0%–100%): How vivid or "colorful" it is. 0% is gray; 100% is the maximum intensity for that hue.
  • Lightness (0%–100%): 0% is black, 100% is white, 50% is the "true" color.
color: hsl(220, 80%, 53%); /* a pleasing, vivid blue */
color: hsl(220, 15%, 45%); /* a muted, corporate slate-gray */
color: hsla(220, 80%, 53%, 0.5); /* same blue, 50% transparent */

For designers and content creators who are not color experts, HSL is magic: once you find a hue you like, you just adjust the other two numbers for variants — lighter backgrounds, darker accents, muted secondary text, and hover states all derive naturally.

Rule of thumb for choosing a format. Use HEX for static brand colors. Use HSL when you are building variants of a color (same hue, different lightness/saturation). Use RGB(A) when you need programmatic control (JavaScript color math, dynamic transparency).

Practical Color Theory for Non-Designers

You do not need a design degree to pick colors that look good together. Start with these proven patterns and you will be ahead of most websites.

1. Pick a single accent hue; derive the rest.

Choose one brand hue (e.g. 220 degrees for blue, 10 degrees for red-orange, 150 degrees for green). Every other color on the site is a variant of that same hue at different lightness and saturation. This guarantees cohesion.

2. Use neutral palettes based on near-gray, not pure gray.

Take your brand hue, drop saturation to 15–20%, and sweep lightness from 90% down to 15%. That produces warm, pleasant grays that look coordinated with the rest of your design. Pure-grayscale grays (#888888, #aaaaaa) read as lifeless.

3. One brightly-saturated color per page — two at most.

High saturation draws the eye. A page saturated everywhere has no focus. Save saturation for buttons, primary calls to action, and key headlines.

4. Keep contrast ratios accessible.

WCAG guidelines require at least 4.5:1 contrast between text and its background for normal-size body text. Tools exist to check this automatically, but as a rough rule, keep your body text above 50% lightness difference from its background in HSL terms.

5. Warm vs. cool temperature affects mood.

Red, orange, and yellow (0–60 degrees) feel warm, energetic, urgent. Green, blue, and purple (120–300 degrees) feel calm, trustworthy, technical. Most design systems anchor on a cool hue (their "resting" color for the interface) and reach for a warm hue for their primary call to action.

Common Color Mistakes and How to Fix Them

  • Using pure black (#000000) for body text. Pure black on pure white produces the highest possible contrast, which can feel harsh and cause halation on bright displays. A near-black like #1e293b is almost as readable and much friendlier.
  • Using pure white (#ffffff) on dark backgrounds. Same problem in reverse — off-whites like #f8fafc feel premium.
  • Too many hues on one page. Every additional hue competes for attention. Three to four hues total is a strong upper bound for most content pages.
  • Ignoring color blindness. Approximately 8% of men have red-green color deficiency. Do not rely on color alone to communicate status — pair colors with icons, labels, and shapes. Test with a color-blindness simulator.
  • Hard-coding brand colors inline instead of using CSS variables. Naming your brand color once (--brand: #2563eb) and reusing the variable everywhere means a one-line change when branding updates.

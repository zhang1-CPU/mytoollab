Text Case Converter: Uppercase, Lowercase, Title Case Explained

Text case seems simple — letters are either uppercase or lowercase, right? But there is a surprising amount of nuance to text casing: title case, sentence case, camel case, snake case, and more. Understanding when to use each case style and how to convert text case quickly can save you hours of manual editing.

Common Text Case Styles Explained

Text casing is about more than just capital letters. Different case styles serve different purposes, from readable body text to programming variable names. Here are the most common case styles you will encounter:

Lowercase

All letters are lowercase. This is the baseline of written text — the default for most body copy and the starting point for most case conversions.

the quick brown fox jumps over the lazy dog

Lowercase is used for body text, paragraphs, and general reading. It is the most readable case for long-form content because our eyes are trained to recognize the shapes of lowercase words.

Uppercase (All Caps)

Every letter is capitalized. Uppercase is attention-grabbing but hard to read in long passages.

THE QUICK BROWN FOX JUMPS OVER THE LAZY DOG

Use uppercase for short text that needs to stand out: buttons, headlines, labels, acronyms, and warning messages. Avoid using all caps for body text — it is significantly harder to read and can come across as shouting.

Title Case (Headline Case)

The first letter of each major word is capitalized. Articles, short prepositions, and conjunctions are usually lowercase unless they are the first or last word.

The Quick Brown Fox Jumps Over the Lazy Dog

Title case is used for — you guessed it — titles and headlines: blog post titles, book titles, article headings, and presentation slide titles. The exact rules for which words to capitalize vary by style guide (AP, Chicago, MLA, etc.).

Sentence Case

Only the first letter of the first word is capitalized, plus any proper nouns. This is how we normally write sentences.

The quick brown fox jumps over the lazy dog

Sentence case is the standard for body text, paragraphs, email bodies, and most regular writing. It is the most natural and readable case for extended text.

Camel Case

Words are joined together without spaces, and each word after the first starts with a capital letter. The shape resembles a camel's humps.

theQuickBrownFoxJumpsOverTheLazyDog

Camel case is primarily used in programming for variable names, function names, and object properties — especially in JavaScript, Java, and C#. Lower camel case (first word lowercase) is the most common variant.

Pascal Case

Like camel case, but the first word is also capitalized.

TheQuickBrownFoxJumpsOverTheLazyDog

Pascal case is commonly used for class names, component names, and type definitions in many programming languages, including C#, Java, and TypeScript.

Snake Case

Words are separated by underscores, and all letters are usually lowercase.

the_quick_brown_fox_jumps_over_the_lazy_dog

Snake case is popular for variable names and function names in Python, Ruby, and PHP, as well as for database column names and API keys.

Kebab Case

Words are separated by hyphens, and all letters are usually lowercase.

Kebab case is commonly used for URLs, CSS class names, HTML attributes, and file names. It is URL-safe and highly readable.

When to Use Each Case Style

Choosing the right case depends on context. Here are some guidelines:

For Writing and Content

  • Sentence case: Body text, paragraphs, email bodies, most regular writing. The default for readability.
  • Title case: Blog post titles, article headings, book titles, presentation titles. Follow your brand's style guide for specific rules.
  • Uppercase: Buttons, short labels, acronyms, warning messages, logos. Use sparingly for maximum impact.
  • Lowercase: Body text (combined with sentence case), casual social media, poetic or stylized text.

For Programming and Development

  • Camel case: Variable names, function names, object properties (JavaScript, Java, C#).
  • Pascal case: Class names, component names, type definitions, React components.
  • Snake case: Python/Ruby/PHP variables, database columns, JSON keys in some APIs.
  • Kebab case: URLs, CSS classes, HTML data attributes, file names.
  • Uppercase snake case (SCREAMING_SNAKE_CASE): Constants, environment variables, configuration values.

Title Case Rules: The Great Debate

Title case seems simple — capitalize the first letter of each word — but the details are surprisingly contentious. Different style guides have different rules about which words to leave lowercase. Here are the most common conventions:

What (Almost) Everyone Agrees On

  • Capitalize the first and last word, always.
  • Capitalize nouns, verbs, adjectives, adverbs, pronouns, and subordinating conjunctions.
  • Lowercase articles (a, an, the) — unless they are the first or last word.

Where Style Guides Disagree

  • Prepositions: Some guides lowercase all prepositions; others capitalize prepositions over a certain length (4 or 5 letters).
  • Coordinating conjunctions: Most lowercase short ones (and, but, or, nor, for, so, yet).
  • The "to" in infinitives: Some lowercase "to" in verb infinitives; others capitalize it.
  • Hyphenated words: Different rules for whether both parts of a hyphenated compound get capitalized.

When in doubt, follow the style guide your organization uses. And when you just need a reasonable default, most title case converters use a sensible set of rules that works for 90% of situations.

Text Case Best Practices for Writers and Developers

For Content Creators

  1. Pick a style and stick with it. Whether you prefer title case or sentence case for headings, be consistent throughout your document or website. Inconsistent casing looks unprofessional.
  2. Use all caps sparingly. All caps is great for short emphasis but terrible for readability in longer passages. Reserve it for buttons, labels, and short warnings.
  3. Know your audience. Different audiences expect different casing conventions. Academic writing follows different rules than social media than technical documentation.
  4. Proofread after conversion. Automatic case converters are great, but they are not perfect. Always proofread after conversion, especially for title case with proper nouns or unusual words.

For Developers

  1. Follow language conventions. Each language has its own casing norms. Use the standard for your language — do not bring Python snake case into a JavaScript codebase.
  2. Be consistent in APIs. If your API uses camel case for JSON keys, use it everywhere. Mixing styles causes confusion and bugs.
  3. Use descriptive names. Casing does not help readability if the names themselves are unclear. A well-named variable in the wrong case is better than a badly named one in the right case.
  4. Use a case converter for refactoring. When you need to rename variables across different systems or convert data formats, an automatic case converter saves a lot of manual find-and-replace work.

Frequently Asked Questions

What is the difference between title case and sentence case?

Title case capitalizes the first letter of most words in a phrase, while sentence case capitalizes only the first letter of the first word (plus proper nouns). Title case is used for titles and headlines, while sentence case is used for regular body text and sentences.

What is camel case used for?

Camel case (writingWordsTogetherLikeThis) is primarily used in programming for variable names, function names, and object properties, especially in languages like JavaScript, Java, and C#. It creates compound words that are both readable and space-efficient without using separators.

Is there a free tool to convert text case?

What are the correct title case rules?

Title case rules vary by style guide, but the general principle is to capitalize major words and lowercase minor words (articles, short prepositions, coordinating conjunctions). Always capitalize the first and last word. The exact rules depend on whether you follow AP, Chicago, MLA, or another style guide.

When should I use uppercase text?

Use uppercase (all caps) sparingly — for short text that needs to grab attention: buttons, labels, acronyms, warning messages, and logos. Avoid using all caps for body text or long passages; it is significantly harder to read and can feel like shouting.

What is the difference between snake_case and kebab-case?

Both separate words with a special character, but snake_case uses underscores and kebab-case uses hyphens. Snake case is common in programming (Python, Ruby, PHP) and databases. Kebab case is common in URLs, CSS class names, and file names where hyphens are more natural or standard.

Can I use a case converter for programming code?

Yes, case converters are great for converting between naming conventions — for example, converting camel case to snake case when working with an API that uses a different style, or converting variable names when porting code between languages. Just make sure to review the results, as converters cannot always perfectly split compound words.
